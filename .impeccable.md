# Impeccable Design Context — Kendo Tournament Builder

## Design Context

### Users
Tournament officials and scorekeepers running live kendo events. Primary context: a laptop on a table courtside at a dojo or sports hall, often in variable lighting (sometimes dim, sometimes harshly lit). The user is focused on the match happening in front of them — the interface must never require them to hunt for the right button. Decisions happen quickly; a wrong tap is friction under pressure.

Secondary context: post-event review on the same laptop, checking bracket results and champion info. Calm, unhurried.

### Brand Personality
**Deliberate · Ceremonial · Precise**

Kendo is a martial art defined by ritual, hierarchy, and disciplined form. The software should carry that character — nothing frivolous, nothing decorative for its own sake. Every element earns its place. The existing palette already captures this perfectly: washi paper (aged warmth), sumi ink (absolute authority), vermillion (the controlled energy of a strike), hakama indigo (structure and formality). Push further into this territory, not away from it.

### Aesthetic Direction
- **Theme**: Light — tournaments run in brightly lit halls; dark mode would be harder to read at a distance. The washi background is the right call.
- **Typography**: Cormorant Garamond for display (italic, editorial weight), Shippori Mincho for any kanji/Japanese elements, Manrope for functional UI copy. This stack is strong — preserve it.
- **Visual language**: Restrained elegance with controlled bursts of vermillion. Ink on paper, not pixels on glass. Avoid glassmorphism, gradients, shadows that compete for attention. Borders and rules should feel like printed lines, not UI chrome.
- **Density**: The scoring views need to be scannable at arm's length. Generous type sizes, high contrast name/score labels, quiet background buttons. The hierarchy is always: **names → scores → controls**.
- **Motion**: Minimal. State changes (score update, match complete) can have a brief fade or scale, but nothing that pulls the eye during a live bout.

### Design Principles
1. **Courtside legibility first** — if a button or score isn't readable at 60cm with ambient hall lighting, it fails. Size and contrast before decoration.
2. **Ceremony over decoration** — visual elements should feel like deliberate ritual choices (the vermillion dot, the ink rule), not UI embellishment.
3. **Hierarchy is fixed** — names are always the most prominent thing; scores second; controls third. Buttons are quiet until needed.
4. **Nothing competes with the match** — the UI is a background instrument. It should never demand attention; it should simply respond when addressed.
5. **Trust the palette** — washi/sumi/shu/indigo is distinctive and cohesive. New elements should extend this system, not introduce new colours.

### Pages & Priority
- **Fullscreen team/singles scoring** — highest priority. Used live, under pressure, often projected or shared on a secondary screen.
- **Match card in bracket** — second. The primary interaction surface during an event.
- **Home page & tournament list** — lower priority. Used calmly before/after events.
- **Setup wizard** — lowest. One-time flow per tournament.
